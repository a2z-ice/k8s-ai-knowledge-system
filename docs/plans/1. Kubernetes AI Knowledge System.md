# Plan 1: Kubernetes AI Knowledge System

**Status:** Complete — All 5 Phases Done
**Date:** 2026-02-22

---

## Overview

A Kubernetes AI Knowledge System that continuously indexes cluster state from a local `kind` cluster into a Qdrant vector database via CDC (Change Data Capture), and exposes a natural-language query interface through n8n using Ollama for embeddings and LLM responses.

---

## Architecture

### Components

| Component | Role | Access |
|-----------|------|--------|
| kind cluster | Source of truth (etcd) | Local |
| Debezium | etcd CDC connector | `localhost:8083` |
| Kafka | Event streaming | `localhost:9092`, topic: `k8s-resources` |
| Qdrant | Vector store | `localhost:6333`, collection: `k8s` |
| n8n | Workflow orchestration | `localhost:5678` (admin/admin) |
| Ollama | Embeddings + LLM | `host.docker.internal:11434` (host machine) |

### Two Parallel Flows

**AI K8s Flow (query pipeline)**
```
User query
  → n8n Chat Trigger
  → Ollama embedding (nomic-embed-text, 768-dim)
  → Qdrant vector search (cosine ≥ 0.75)
  → retrieved docs as context
  → Ollama LLM (chat model)
  → structured table response
```

**CDC Flow (sync pipeline)**
```
kind etcd change
  → Debezium etcd connector
  → Kafka topic: k8s-resources
  → n8n Kafka trigger
  → Ollama embedding (nomic-embed-text)
  → delete Qdrant point by resource_uid
  → insert updated vector
```

### Qdrant Point Schema

Collection: `k8s` | Dimensions: 768 | Distance: Cosine

```json
{
  "resource_uid":          "<uid>",
  "kind":                  "<Kind>",
  "namespace":             "<namespace>",
  "name":                  "<name>",
  "labels":                { ... },
  "annotations":           { ... },
  "raw_spec_json":         "{ ... }",
  "last_updated_timestamp": "<ISO8601>"
}
```

## Implementation Phases

### Phase 1 — Infrastructure Setup
1. Create kind cluster: `kind create cluster --name k8s-ai-classic`
2. Verify etcd is accessible from Debezium
3. Start Docker Compose services: `docker compose -f docker-compose-k8s-classic-ai-system.yml up -d`
4. Pull Ollama models on host:
   - `ollama pull <chat-model>`
   - `ollama pull nomic-embed-text`
5. Create Qdrant collection: `curl -X PUT http://localhost:6333/collections/k8s -d @qdrant_k8s_collection_schema.json`

### Phase 2 — CDC Flow Implementation
1. Register Debezium etcd connector: `curl -X POST http://localhost:8083/connectors -d @debezium_k8s_connector.json`
2. Verify Kafka topic `k8s-resources` receives events
3. Import `classic_n8n_cdc_k8s_flow.json` via n8n UI
4. Verify CDC flow: create a namespace in kind → confirm Qdrant point inserted

### Phase 3 — AI Flow Implementation
1. Import `classic_n8n_ai_k8s_flow.json` via n8n UI
2. Paste `Claude_Prompt_Companion_K8s_AI.md` into the AI Chat Agent system prompt node
3. Test query: "List all namespaces" → expect structured table response

### Phase 4 — E2E Testing
```
npx playwright test playwright_k8s_ai_e2e.spec.ts
```

Test scenarios:
- Create namespace → verify Qdrant insertion
- Update deployment → verify vector replacement (dedup via `resource_uid`)
- Delete service → verify point removal
- Ask "how many namespaces?" → validate structured table output

### Phase 5 — Reset REST Endpoint
Provides a single HTTP call that wipes the Qdrant vector database and triggers a full re-sync from the live Kubernetes cluster.

**Components modified:**
- `k8s-watcher/watcher.py` — Added HTTP server on port 8080 with:
  - `GET /healthz` → `{"status":"ok"}`
  - `POST /resync` → lists all 9 resource types via K8s API, publishes `ADDED` events to Kafka (202 Accepted, runs in background thread)
- `docker-compose.yml` — Exposed k8s-watcher port `8085:8080` (host:container)
- `workflows/classic_n8n_reset_k8s_flow.json` — New 5-node n8n workflow:
  1. **Reset Webhook** — `POST /webhook/k8s-reset` (responseMode: lastNode)
  2. **Delete Qdrant Collection** — `DELETE http://qdrant:6333/collections/k8s`
  3. **Recreate Qdrant Collection** — `PUT http://qdrant:6333/collections/k8s` (768-dim Cosine)
  4. **Trigger Resync** — `POST http://k8s-watcher:8080/resync`
  5. **Format Response** — returns `{status, message, reset_at}`

**Usage:**
```bash
curl -X POST http://localhost:5678/webhook/k8s-reset
# Returns: {"status":"ok","message":"Qdrant collection cleared and k8s-watcher resync triggered...","reset_at":"<ISO8601>"}
# After ~30-60 s, Qdrant repopulates with all current cluster resources
```

**Workflow ID:** `JItVx5wVu0WTIvkA`

**E2E test added:** Test 5 in `tests/e2e/playwright_k8s_ai_e2e.spec.ts`
- Calls reset endpoint, verifies `status=ok`
- Asserts Qdrant is empty (0 points) immediately after reset
- Waits up to 90 s for CDC resync to repopulate ≥ 10 points

## Operational Notes

- **Deduplication:** CDC flow always deletes by `resource_uid` before insert — idempotent for create/update/delete events.
- **Similarity threshold:** AI flow filters Qdrant results to cosine ≥ 0.75 to avoid hallucination on low-confidence matches.
- **Ollama on host:** All Docker services reach Ollama via `host.docker.internal:11434`. Do not run Ollama inside Docker.
- **Retry logic:** Add retry handling in n8n Kafka consumer node for transient failures.
