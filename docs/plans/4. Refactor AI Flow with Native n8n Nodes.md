# Plan 4: Refactor All Flows with Native n8n AI Nodes

## Context

The current AI_K8s_Flow and CDC_K8s_Flow use raw HTTP Request nodes to call Ollama and Qdrant. n8n ships built-in LangChain nodes that handle this natively. This plan replaces all manual HTTP-based AI/embedding/vector operations with n8n's native nodes across both flows.

---

## Critical Pre-Requisite: `pageContent` in Qdrant Payload

n8n's LangChain Qdrant retriever (in the AI flow) and inserter (in CDC flow) both use `pageContent` as the document text field. Currently our Qdrant points have no `pageContent` field — the embed text was never stored in the payload.

**All paths forward require** adding `pageContent: embed_text` to Qdrant points (either via the CDC code node or natively via the Qdrant insert node's content field option). A `/reset-db` after deployment repopulates Qdrant with the new structure.

## Step 3: Rewrite AI_K8s_Flow

**File:** `workflows/n8n_ai_k8s_flow.json`

**Before:** 6-node linear chain (Chat Trigger → Generate Embedding [HTTP] → Qdrant Search [HTTP] → Build Prompt [Code] → LLM Chat [HTTP] → Format Response [Code])

**After:** 5 nodes (Chat Trigger + AI Agent + 3 sub-nodes wired via LangChain slots)

### Nodes

**Chat Trigger** — unchanged: `@n8n/n8n-nodes-langchain.chatTrigger`, typeVersion 1, `webhookId: k8s-ai-chat`, `public: true`

**AI Agent** — `@n8n/n8n-nodes-langchain.agent`, typeVersion 1.7
- `agentType: toolsAgent`
- System message: expert Kubernetes AI assistant, always use kubernetes_knowledge_base tool, markdown tables for structured data

**Ollama Chat Model** — `@n8n/n8n-nodes-langchain.lmChatOllama`, typeVersion 1
- `model: qwen3:8b`, `options.temperature: 0.1`
- Credential: `ollamaApi` → `id: ollama-local`
- Connection: `ai_languageModel` → AI Agent

**Qdrant Vector Store** — `@n8n/n8n-nodes-langchain.vectorStoreQdrant`, typeVersion 1
- `mode: retrieve-as-tool`
- `qdrantCollection: { __rl: true, value: "k8s", mode: "id" }`
- `toolName: kubernetes_knowledge_base`
- `toolDescription: "Search the Kubernetes cluster knowledge base..."`
- `options.limit: 30`
- Credential: `qdrantApi` → `id: qdrant-local`
- Connection: `ai_tool` → AI Agent

**Embeddings Ollama** — `@n8n/n8n-nodes-langchain.embeddingsOllama`, typeVersion 1
- `model: nomic-embed-text:latest`
- Credential: `ollamaApi` → `id: ollama-local`
- Connection: `ai_embedding` → Qdrant Vector Store

## Step 4: Update CDC_K8s_Flow — Native Qdrant Insert

**File:** `workflows/n8n_cdc_k8s_flow.json`

**Nodes 1–4 unchanged:** Kafka Trigger → Parse Message → Delete Existing Vector (HTTP) → Is Delete Event?

**Replace nodes 5–7** with:

**Node 5 (new): "Format Document" Code node**
- Builds `{ pageContent: embed_text, metadata: { all fields } }`

**Node 6 (new): "Insert to Qdrant" vectorStoreQdrant**
- `mode: insert`, `qdrantCollection: k8s`
- Credential: `qdrantApi` → `id: qdrant-local`

**Node 7 (new sub-node): "Embeddings Ollama (CDC)" embeddingsOllama**
- `model: nomic-embed-text:latest`
- Credential: `ollamaApi` → `id: ollama-local`
- Connection: `ai_embedding` → Insert to Qdrant

Note: **Delete Existing Vector** HTTP node is kept — no native delete-by-ID LangChain node exists.

## Risk Notes

- **qwen3:8b tool-calling**: Required for `toolsAgent`. qwen3:8b supports Ollama function calling. If tool calls fail, change `agentType` to `conversationalAgent`.
- **Qdrant insert custom ID**: If the running n8n version's Qdrant node lacks a custom ID option, the CDC insert falls back to HTTP nodes. The AI flow is unaffected.
- **Credential IDs must exist before activation**: `setup.sh` creates `ollama-local` and `qdrant-local` credentials while n8n is scaled to 0, before reimporting + activating workflows.
