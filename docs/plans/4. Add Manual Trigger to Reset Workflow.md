## How n8n Handles Multiple Triggers

In n8n, multiple trigger nodes are valid in a single workflow. Each trigger independently connects to the same downstream node via the `connections` object — no Merge node is needed. When the webhook fires, only that execution path runs. When the Manual Trigger fires (via UI or REST API), only that path runs.

## Key Technical Findings

### Why not `n8n execute` CLI?

`n8n execute --id k8sRSTflow00001` (via `kubectl exec`) always tries to bind port 5679 (Task Broker), conflicting with the already-running n8n server. No env override works:
- `N8N_TASK_BROKER_PORT=5680` — ignored
- `N8N_RUNNERS_ENABLED=false` — still fails

### Why not browser click via Playwright?

n8n overlays execution results on the canvas after a workflow runs. This overlay has a higher z-index and captures all pointer events — even `force: true` and `dispatchEvent('click')` don't reach the underlying Vue event handler. The workflow never executes (confirmed: no new `mode=manual` executions appeared).

### n8n REST API — correct request format (discovered via network interception)

```
POST /rest/workflows/{id}/run
{
  "workflowData": { ...full workflow JSON from GET /rest/workflows/{id} as .data... },
  "startNodes": [],
  "triggerToStartFrom": { "name": "Manual Trigger" }
}
```

- `triggerToStartFrom.name` must match the node's `name` field exactly (not `id`)
- `workflowData` must include the `id` field and full node/connection data
- `startNodes: []` (empty array, not null)
- Wrong field name `forcedTriggerNode` causes: `Cannot read properties of null (reading 'nodeName')`
- Omitting `workflowData` causes: `Cannot read properties of undefined (reading 'id')`

Returns: `{ "data": { "executionId": "1174" } }` with HTTP 200.

Poll `GET /rest/executions/{executionId}` — response `.data.mode === "manual"` and `.data.status === "success"`.

### Qdrant repopulation timing

After the execution completes (`status=success`), the Qdrant collection has already been deleted and recreated (synchronous nodes), and `/resync` has been called (async, returns 202). Repopulation takes < 5 seconds in the test environment.
