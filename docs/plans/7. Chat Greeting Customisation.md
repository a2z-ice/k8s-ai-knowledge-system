# 7. Chat Greeting Customisation

**Date:** 2026-03-07
**Status:** Complete

---

## Overview

The n8n AI Chat interface (`/webhook/k8s-ai-chat/chat`) ships with a default greeting:

> Hi there!
> My name is Nathan. How can I assist you today?

This was changed to a project-specific greeting that identifies the assistant by name and the target cluster.

---

## What Changed

### Before

| Setting | Value |
|---------|-------|
| Title | *(n8n default)* |
| Subtitle | *(none)* |
| Greeting | Hi there! / My name is Nathan. How can I assist you today? |

### After

| Setting | Value |
|---------|-------|
| Title | K8s AI Assistant |
| Subtitle | Ask me anything about your Kubernetes cluster |
| Greeting | Hi there! / My name is Assad. I am k8s assistant for the kind cluster k8s-ai. How can I assist you today? |

---

## How It Works

The greeting is configured in the **Chat Trigger** node parameters inside `workflows/n8n_ai_k8s_flow.json`.

### Relevant JSON (Chat Trigger node)

```json
{
  "parameters": {
    "public": true,
    "initialMessages": "Hi there!\nMy name is Assad. I am k8s assistant for the kind cluster k8s-ai. How can I assist you today?",
    "options": {
      "title": "K8s AI Assistant",
      "subtitle": "Ask me anything about your Kubernetes cluster"
    }
  },
  "id": "chat-trigger",
  "name": "When chat message received",
  "type": "@n8n/n8n-nodes-langchain.chatTrigger",
  "typeVersion": 1.4
}
```

### Key fields

| Field | Location | Description |
|-------|----------|-------------|
| `initialMessages` | Top-level parameter (NOT inside `options`) | Newline-separated lines. Each `\n` becomes a separate chat bubble. |
| `options.title` | Inside `options` | Bold title shown at the top of the chat widget. |
| `options.subtitle` | Inside `options` | Smaller text shown below the title. |

> **Important:** `initialMessages` must be a **top-level parameter**, not nested inside `options`. Placing it inside `options` has no effect — n8n falls back to the default "Nathan" greeting.

## How to Customise

### Step 1 — Edit the workflow JSON

Open `workflows/n8n_ai_k8s_flow.json` and modify the `initialMessages` field in the chat-trigger node:

```json
"initialMessages": "Your custom greeting line 1\nYour custom greeting line 2"
```

Each `\n` creates a new chat bubble in the UI.

### Step 2 — Reimport and deploy

```bash
# Get the n8n pod name
N8N_POD=$(kubectl --context kind-k8s-ai -n k8s-ai get pod -l app=n8n -o jsonpath='{.items[0].metadata.name}')

# Copy the updated workflow into the pod
kubectl --context kind-k8s-ai -n k8s-ai cp workflows/n8n_ai_k8s_flow.json ${N8N_POD}:/tmp/

# Import and publish
kubectl --context kind-k8s-ai -n k8s-ai exec ${N8N_POD} -- \
  su -s /bin/sh node -c "n8n import:workflow --input=/tmp/n8n_ai_k8s_flow.json"
kubectl --context kind-k8s-ai -n k8s-ai exec ${N8N_POD} -- \
  su -s /bin/sh node -c "n8n publish:workflow --id=k8sAIflow000001"

# Restart n8n for changes to take effect
kubectl --context kind-k8s-ai -n k8s-ai rollout restart deployment/n8n
kubectl --context kind-k8s-ai -n k8s-ai rollout status deployment/n8n --timeout=60s
```

### Step 3 — Verify

Wait ~15 seconds after restart, then open:

```
http://localhost:30000/webhook/k8s-ai-chat/chat
```

Or verify via CLI:

```bash
curl -s http://localhost:30000/webhook/k8s-ai-chat/chat | grep 'initialMessages'
```

Expected output:

```
initialMessages: ["Hi there!","My name is Assad. I am k8s assistant for the kind cluster k8s-ai. How can I assist you today?"],
```

---

## Limitations

- **Static only** — The `initialMessages` field is rendered as static HTML before any workflow logic executes. It does not support n8n expressions (`={{ }}`) or runtime lookups (e.g., dynamic cluster name from the Kubernetes API).
- **Cluster name must be updated manually** — If the kind cluster is renamed (e.g., from `k8s-ai` to something else), update the `initialMessages` string in the workflow JSON and redeploy.
