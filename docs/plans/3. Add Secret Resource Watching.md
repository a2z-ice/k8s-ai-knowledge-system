# Plan 3: Add Secret Resource Watching to k8s-watcher

## Context

**Problem:** Querying "give me secrets information of each namespace" returns "No indexed Kubernetes
resources found in vector database." The k8s-watcher only watched 9 resource types; `Secret` was
absent — so no Secret events were ever published to Kafka, and Qdrant had no Secret vectors.

**Secondary issue:** `obj.kind` is always None from the k8s watch stream AND `raw.get("kind", "")`
is also empty — embed_text read "Kubernetes  named ..." (empty kind). The `kind_hint` parameter
fixes this for all resource types.

**Security constraint:** Secret `.data` values (base64-encoded passwords/tokens) must NEVER appear
in the Qdrant payload or embed_text. Only safe metadata is stored: name, namespace, labels, type,
and the list of key names (not values).

## Implementation

### watcher.py changes

1. `obj_to_payload(event_type, obj, kind_hint="")`:
   - `kind = obj.kind or raw.get("kind", "") or kind_hint`
   - Secret-safe spec: `{"type": raw.get("type"), "dataKeys": list(raw.get("data", {}).keys())}`

2. `watch_stream` and `resync_all` both pass `kind_hint=label` to `obj_to_payload`.

3. Added `(v1.list_secret_for_all_namespaces, "Secret")` to both `resource_fns` in `resync_all`
   and `watchers` in `main()` — 10 resource types total (was 9).

### RBAC change

Added `"secrets"` to the core API group resources list.

### E2E tests

- **Test 6** — CDC Secret: creates `e2e-test-secret` in `k8s-classic-ai`, verifies Kafka offset advances,
  upserts safe metadata to Qdrant, asserts `kind=Secret`, asserts raw values not stored.
- **Test 7** — AI secrets query: embeds "What secrets exist in the kube-system namespace?",
  searches Qdrant (threshold 0.3), asserts ≥1 result with `kind=Secret` and
  `namespace=kube-system`, calls Ollama LLM, asserts response mentions "secret" and "kube-system"
  but does NOT contain raw values.
