# Plan 5: Add PostgreSQL Chat Memory + pgAdmin to AI K8s Flow

## Context

The AI_K8s_Flow currently has no conversation memory — every webhook call is stateless. This plan adds PostgreSQL-backed chat memory via n8n's native `MemoryPostgresChat` node, so the AI agent can recall previous messages within an ongoing session. A dedicated `Memory_Clear_Flow` workflow handles both automatic (hourly) and manual clearing of all chat history. pgAdmin is added as a browser-accessible UI to inspect/manage the `n8n_chat_histories` table.

## Architecture

```
AI_K8s_Flow:
  Chat Trigger → AI Agent ←[ai_memory]── Postgres Chat Memory (postgres:5432, k8s-ai-global session)
                     ↓                                           ↓
               Qdrant Vector Store                  n8n_chat_histories table
               Ollama Chat Model

Memory_Clear_Flow:
  Manual Trigger  ──→
                      DELETE FROM n8n_chat_histories
  Schedule Trigger ─→ (every hour)
```

## Memory Configuration

- **Session key**: `k8s-ai-global` (fixed — all queries share one persistent session)
- **Context window**: 5 message pairs (last 5 exchanges kept as context)
- **Table**: `n8n_chat_histories` (auto-created by n8n on first use)
- **Auto-clear**: hourly via Memory_Clear_Flow Schedule Trigger
