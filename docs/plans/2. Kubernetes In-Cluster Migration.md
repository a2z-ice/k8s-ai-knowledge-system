# Plan 2: Kubernetes In-Cluster Migration

## Goal

Migrate all services from Docker Compose into the existing `k8s-ai` kind cluster. Expose n8n at `n8n.genai.prod:30000` via NodePort. Ollama stays on the host. Debezium is removed entirely. Existing `./data/` volume data is preserved via kind `extraMounts` + hostPath PVs.

## Key Design Decisions

- **kind `extraMounts`**: `./data` → `/data` in node; hostPath PVs reference `/data/{n8n,qdrant,kafka}`
- **Static PV binding**: `storageClassName: ""` + `claimRef` pre-binds each PV to its specific PVC
- **k8s-watcher RBAC**: ServiceAccount + ClusterRole with list/watch on 9 resource types
- **In-cluster config**: watcher detects `KUBERNETES_SERVICE_HOST` env var and uses `load_incluster_config()`
- **n8n hostAliases**: maps `host.docker.internal` → `192.168.1.154` so workflow URLs are unchanged
- **n8n initContainer**: `chown -R 1000:1000` fixes ownership from Docker-written files
- **Kafka KRaft**: `KAFKA_CONTROLLER_QUORUM_VOTERS` uses stable StatefulSet pod DNS
- **Same CLUSTER_ID**: preserves existing Kafka data in `./data/kafka`
- **Debezium**: service removed from `docker-compose.yml`; connector JSON kept for reference

## Deployment Sequence

```bash
# 0. Stop Docker Compose
docker compose -f docker-compose.yml down

# 1. Recreate kind cluster
kind delete cluster --name k8s-ai
kind create cluster --config infra/kind-config.yaml

# 2. Namespace + PVs
kubectl --context kind-k8s-ai apply -f infra/k8s/00-namespace.yaml
kubectl --context kind-k8s-ai apply -f infra/k8s/01-pvs.yaml

# 3. Kafka
kubectl --context kind-k8s-ai apply -f infra/k8s/kafka/
kubectl --context kind-k8s-ai -n k8s-ai rollout status statefulset/kafka --timeout=120s

# 4. Qdrant
kubectl --context kind-k8s-ai apply -f infra/k8s/qdrant/
kubectl --context kind-k8s-ai -n k8s-ai rollout status deployment/qdrant --timeout=60s

# 5. k8s-watcher (build + load image first)
docker build -t k8s-watcher:latest ./k8s-watcher/
kind load docker-image k8s-watcher:latest --name k8s-ai
kubectl --context kind-k8s-ai apply -f infra/k8s/k8s-watcher/
kubectl --context kind-k8s-ai -n k8s-ai rollout status deployment/k8s-watcher --timeout=90s

# 6. n8n
kubectl --context kind-k8s-ai apply -f infra/k8s/n8n/
kubectl --context kind-k8s-ai -n k8s-ai rollout status deployment/n8n --timeout=120s

# 7. Import + activate workflows (see CLAUDE.md for full commands)

# 8. /etc/hosts: 192.168.1.154 n8n.genai.prod
```

## Known Risks

| Risk | Mitigation |
|------|-----------|
| Kafka refuses to start with existing `./data/kafka` | Delete `./data/kafka/*` and trigger `/reset-db` after pods up |
| n8n SQLite owned by root | initContainer `chown -R 1000:1000` before n8n starts |
| macOS Docker Desktop may not share `/Volumes/Other/...` | Verify with test mount before cluster creation |
| Workflow IDs change after re-import on fresh DB | Run `kubectl exec n8n-pod -- n8n list:workflow` to discover new IDs |
